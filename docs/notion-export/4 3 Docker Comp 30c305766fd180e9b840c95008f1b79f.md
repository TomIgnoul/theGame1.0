# 4.3 Docker Comp

## 4.3.1 Doel

De `docker-compose.yml` definiГ«ren als concrete runtime-implementatie van het platform design (4.2), met focus op **services** en **persistente data**.

## 4.3.2 Services (Core)

### `postgres`

- Role: persistente datastore
- Persistent: **ja** (volume)
- Consumers: `backend`

### `backend`

- Role: FastAPI BFF API
- Persistent: **nee** (state in Postgres)
- Dependencies: `postgres`

### `n8n`

- Role: workflow runner (must)
- Persistent: **ja** (n8n config/workflows storage)
- Dependencies: none (kan later backend callen voor checks)

## 4.3.3 Volumes (Core)

### `pgdata`

- Purpose: Postgres data persistent maken over restarts
- Bound to: `postgres`

### `n8n_data`

- Purpose: n8n state/config persistent maken
- Bound to: `n8n`

## 4.3.4 Compose Ownership Rules

- `docker-compose.yml` is runtime-config, maar **canonieke acties blijven scripts** (`./scripts/*`).
- n8n workflows mogen scripts triggeren, maar scripts blijven source of truth.

## 4.3.5 Out of scope (deze sectie)

- Ports mapping (komt in 4.3.6)
- Env vars detail (komt in 4.3.6)
- Healthchecks (komt in 4.3.6)

### Ports (Host exposure policy)

**Principes**

- Alleen exposen wat nodig is voor lokale development en verificatie.
- Frontend draait buiten compose.

**`postgres`**

- Host port: **5432**
- Reden: lokale inspectie/debugging + tooling (praktisch voor development).

**`backend`**

- Host port: **8000**
- Reden: frontend dev server en test-calls (curl) moeten backend bereiken.

**`n8n`**

- Host port: **5678**
- Reden: n8n UI moet bereikbaar zijn voor het team.

**Principes**

- `.env.example` bevat placeholders; echte `.env` is lokaal en gitignored.
- Backend doet **fail-fast** op ontbrekende вЂњrequiredвЂќ env vars (per milestone).
