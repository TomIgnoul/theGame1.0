# 3.5 UI Design

## 1) Screens & Navigation (MVP)

### Base screen

- **S1 Home/Map** = map is het anker en blijft zichtbaar (single-page).

### Overlays (stack order)

1. **S3 Route Builder** (panel/bottom sheet)
2. **S2 POI Detail** (full-screen)
3. **Chat Drawer** (overlay boven Route View)

### Navigation rules

- Tap pin → open **S2 POI Detail** (full-screen)
- In **S2**: CTA “Build route” → **sluit S2** → open **S3** met theme prefilled
- In **S3**: “Generate route” success → toon **S4 Route View**
- In **S4**: tap stop / chat bubble → open **Chat Drawer**
- Chat Drawer header is altijd: **`Stop {n} - {POI naam}`**

---

## 2) Naming & Copy Rules (strict)

- Header format (altijd): **`Stop {n} - {POI naam}`**
- Empty pins message: **“Geen resultaten”**
- Practical info missing label: **“Praktische info niet gevonden”**
- Rate limit message (429): **“Even wachten en opnieuw proberen.”**
- Max chat message length: **500 chars**

---

## 3) State Machines (MVP)

### S1 Home/Map - State machine

**States**

- `idle` (pins zichtbaar)
- `loading_pins`
- `error_pins`
- `empty_pins`

**Transitions**

- `idle` → `loading_pins` (app load / refresh / filter change)
- `loading_pins` → `idle` (pins ontvangen)
- `loading_pins` → `empty_pins` (0 pins)
- `loading_pins` → `error_pins` (API error)
- `error_pins` → `loading_pins` (retry)
- `empty_pins` → `loading_pins` (filter aanpassen)

**UI rules**

- Active filter badge zichtbaar wanneer filter actief is
- Empty message: “Geen resultaten”

---

### S2 POI Detail (full-screen) - State machine

**States**

- `loading_detail`
- `detail_ready` (zonder story)
- `story_loading`
- `story_ready`
- `story_error` *(incl. 429)*

**Transitions**

- open detail: `loading_detail` → `detail_ready`
- click “Generate story”: `detail_ready` → `story_loading`
- `story_loading` → `story_ready` (answer ontvangen)
- `story_loading` → `story_error` (error of 429)
- `story_error` → `story_loading` (retry)

**UI rules**

- Practical info:
    - als ontbreekt: label “Praktische info niet gevonden”
- Button “Generate story”:
    - disabled tijdens `story_loading`

---

### S3 Route Builder (panel/bottom sheet) - State machine

**States**

- `builder_open`
- `input_invalid`
- `route_loading`
- `route_ready`
- `route_error` (generic)
- `route_no_possible` (404 no-route)

**Transitions**

- open builder: `builder_open`
- invalid input: `builder_open` ↔ `input_invalid`
- click “Generate route”:
    - valid → `route_loading`
    - invalid → `input_invalid`
- `route_loading` → `route_ready` (success)
- `route_loading` → `route_no_possible` (404)
- `route_loading` → `route_error` (other errors)

**UI rules**

- Validatie: min één van km/min ingevuld
- `route_no_possible`: toon suggestie (tijd/km verhogen of ander thema)

---

### S4 Route View - State machine

**States**

- `route_rendering`
- `route_ready`
- `start_sending`
- `start_ok`
- `start_error`

**Transitions**

- `route_rendering` → `route_ready` (polyline + stops getekend)
- click “Start walk”: `route_ready` → `start_sending`
- `start_sending` → `start_ok` (ok=true) → terug naar `route_ready`
- `start_sending` → `start_error` (retry)

**UI rules**

- markers genummerd (Stop 1..n)
- stops lijst matcht marker volgorde

---

### Chat Bubble + Drawer (over Route View) - State machine

**States**

- `closed`
- `open_idle`
- `sending`
- `answer_ready`
- `rate_limited` (429)
- `chat_error`

**Transitions**

- open chat: `closed` → `open_idle` (header: `Stop {n} - {POI naam}`)
- send msg: `open_idle`/`answer_ready` → `sending`
- `sending` → `answer_ready` (answer)
- `sending` → `rate_limited` (429)
- `sending` → `chat_error` (other error)
- retry: `rate_limited`/`chat_error` → `sending`
- close: any → `closed`

**UI rules**

- Header verplicht: `Stop {n} - {POI naam}`
- Max message length 500 chars
- 429 message: “Even wachten en opnieuw proberen.”

- Button: **Generate story**

- Floating chat bubble (visible in Route View)

    - Header: **`Stop {n} - {POI naam}`**

## 5) S5 Admin Portal (AN-40)

### Screen & Navigation

- Route: **`/admin`**
- Open `/admin` → check backend-issued admin session cookie
- No valid session → show **Admin Login**
- Valid session → show **Admin Portal**
- Logout → clear session via backend and return to login screen

### State machine

**States**

- `session_checking`
- `login_ready`
- `login_submitting`
- `dashboard_loading`
- `dashboard_ready`
- `dashboard_empty`
- `dashboard_error`
- `dashboard_unauthorized`
- `logout_pending`

**Transitions**

- open `/admin`: `session_checking` → `dashboard_loading` (valid session) or `dashboard_unauthorized`
- `dashboard_unauthorized` → `login_ready`
- submit passphrase: `login_ready` → `login_submitting`
- `login_submitting` → `dashboard_loading` (success)
- `login_submitting` → `login_ready` (invalid passphrase / auth unavailable)
- `dashboard_loading` → `dashboard_ready` (all analytics payloads loaded and `hasData=true`)
- `dashboard_loading` → `dashboard_empty` (overview, timeseries, and breakdowns all return `hasData=false`)
- `dashboard_loading` → `dashboard_error` (non-auth backend error)
- any analytics `401` → `dashboard_unauthorized`
- click logout: `dashboard_ready` / `dashboard_empty` / `dashboard_error` → `logout_pending` → `login_ready`

### Must components

- **Admin Login**
  - Title: **Analytics access**
  - Password field: **Admin passphrase**
  - Button: **Open dashboard**
  - Inline auth error message
- **Admin Portal**
  - Header: **Admin analytics portal**
  - Button: **Logout**
  - Filter bar:
    - From date
    - To date
    - Theme select
    - Button: **Apply filters**
  - KPI cards:
    - Route generations
    - Route starts
    - Route failures
    - POI detail views
    - Chat sends
  - Trend chart:
    - Route generations vs starts
  - Breakdown charts:
    - Theme breakdown
    - POI breakdown
  - Loading state card
  - Empty state card
  - Error state card + Retry

### UI rules

- Portal blijft strikt **read-only**
- Frontend praat alleen met backend endpoints:
  - `POST /api/admin/auth/login`
  - `POST /api/admin/auth/logout`
  - `GET /api/admin/analytics/overview`
  - `GET /api/admin/analytics/timeseries`
  - `GET /api/admin/analytics/breakdowns`
- Filter submit refresht KPI, timeseries, en breakdown data met exact dezelfde `from` / `to` / `theme` set
- Grafieken blijven simpel en voorspelbaar:
  - één compacte trend chart voor route generations vs starts
  - horizontale breakdown bars voor theme en POI
- Empty state toont altijd het actieve datumbereik en actieve theme-filter
- Unauthorized state toont nooit analytics data

## 6) Done Criteria (UI)

UI is “done” when every screen:

- has loading/empty/error states wired
- matches acceptatiecriteria bewijs (badge, stop header, no practical info hallucination)
- works on mobile (touch-first)
