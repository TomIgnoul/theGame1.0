3) Routes (stateless)
POST /api/routes
Doel: route genereren op basis van user input (stateless; geen verplichte opslag van routes in DB).
Request
{  "theme": "Art",  "kmTarget": 7,  "shape": "loop",  "start": { "lat": 50.8466, "lng": 4.3528 },  "end": null}
Validatie
theme verplicht, moet in whitelist.
kmTarget verplicht, binnen 1–15 km.
shape verplicht: "loop" of "a_to_b".
start verplicht:
bij "loop": start definieert begin/eindgebied.
bij "a_to_b": end verplicht met lat en lng.
Backend mag 6–10 gems selecteren en tot 2 retries doen om binnen ±10% van kmTarget te blijven.
200 OK
{  "shape": "loop",  "kmTarget": 7,  "kmResult": 6.8,  "gems": [    { "id": "uuid", "title": "..." }  ],  "polyline": "encoded_polyline_string",  "warnings": []}
Notes
warnings kan zaken bevatten zoals “INSUFFICIENT_GEMS” of hints om criteria te versoepelen.
Route is stateless in de zin dat de client met deze payload alles heeft wat nodig is voor de kaartweergave; optionele logging kan via route_logs (zie 3.2 DB Design).
Errors
400 invalid input (km buiten bounds, shape ongeldig, ontbrekende start/end, etc.)
404 no route possible (criteria te strikt / onvoldoende gems)
429 rate limited
502 upstream routing failure (Google error)
500 unexpected
4) Story (AI)
POST /api/gems/:id/story
Doel: AI-story genereren of uit cache halen voor een specifieke Gem.
Request
{  "theme": "Art",  "language": "en"}
Validatie
theme verplicht, in whitelist.
language verplicht (bv. "en" of "nl").
Eventuele extra velden (bv. user prompt) worden afgekapt op max 500 chars.
200 OK
{  "gemId": "uuid",  "theme": "Art",  "language": "en",  "promptVersion": "v1",  "storyText": "..."}
Notes
Backend haalt Gem-context op (title, theme, locatie, praktische info) en bouwt de daadwerkelijke LLM-prompt.
Story wordt opgeslagen in gem_stories met unieke sleutel (gemId, theme, language, promptVersion) zodat herhaalde requests uit cache kunnen komen.
Errors
400 invalid input (ongeldige theme/language, overschreden maxlength, etc.)
404 gem not found
429 rate limited (te veel story requests)
502 upstream AI failure
503 generation timeout
500 unexpected
Privacy rule
Story-input/-output wordt niet als chatlog opgeslagen; alleen de finale story wordt gecachet (zonder raw user conversation).
5) Analytics ingest
POST /api/analytics/events
Doel: allowlisted frontend analytics-events veilig doorgeven naar dedicated analytics storage zonder chatcontent of arbitraire metadata.
Request
{  "eventType": "filter_applied" | "route_started" | "stop_chat_opened",  "theme": "Culture",  "poiId": "uuid",  "metadata": { "shape": "loop", "kmTarget": 7, "kmResult": 6.8, "gemCount": 8 } }
Validatie
eventType verplicht en moet in frontend allowlist staan.
Onbekende event types geven 400.
Alleen goedgekeurde velden per event zijn toegestaan.
Geen arbitraire metadata blobs.
Geen chat bodies, geen AI-antwoorden, geen request bodies, geen directe PII.
202 Accepted
{  "accepted": true }
Errors
400 invalid event type or invalid payload
Notes
Backend capture schrijft route_generated, route_generation_failed, poi_detail_opened, story_generated en stop_chat_sent intern weg.
Frontend ingest is enkel voor allowlisted UI-triggered analytics-events.
Analytics write failures mogen de primaire user flow niet blokkeren; backend logt enkel metrics-only en gaat verder.
6) Admin auth
POST /api/admin/auth/login
Doel: minimale v1 admin-auth zonder accounts; backend valideert gedeelde env-passphrase en geeft een httpOnly sessie-cookie uit.
Request
{  "passphrase": "string" }
200 OK
{  "ok": true }
Errors
400 missing passphrase
401 invalid admin passphrase
503 admin auth unavailable
Notes
Cookie is HttpOnly en SameSite=Lax.
Cookie is Secure in https/prod-achtige omgevingen en niet bedoeld om door frontend-JavaScript uitgelezen te worden.
POST /api/admin/auth/logout
Doel: admin-sessie expliciet wissen.
200 OK
{  "ok": true }

theme optional en moet, indien opgegeven, in whitelist staan.

9) Admin analytics breakdowns
GET /api/admin/analytics/breakdowns?from=YYYY-MM-DD&to=YYYY-MM-DD&theme={theme?}
Doel: breakdowns per theme en per POI voor read-only analytics visualisaties.
200 OK
{  "filters": { "from": "2026-04-01", "to": "2026-04-07", "theme": null },  "hasData": true,  "themeBreakdown": [    { "theme": "Culture", "totalEvents": 14, "routeGenerations": 5, "routeStarts": 4, "routeFailures": 1, "filterApplies": 2, "poiDetailViews": 1, "storyGenerations": 0, "chatOpens": 0, "chatSends": 1 }  ],  "poiBreakdown": [    { "poiId": "uuid", "title": "Grand Place", "theme": "Culture", "totalEvents": 9, "poiDetailViews": 3, "storyGenerations": 2, "chatOpens": 2, "chatSends": 2 }  ] }

400 invalid analytics filters
401 admin auth required
500 analytics breakdowns unavailable
Notes
Alle admin analytics endpoints zijn read-only.
Lege datasets blijven 200 OK met `hasData=false`, lege arrays en/of zero-value payloads zodat de frontend een stabiele empty state kan tonen.
Done criteria (API contract)
UI kan alle kernflows uitvoeren met deze endpoints:
Gems ophalen + filteren voor de kaart.
Gem detail laden met correcte fallbacks (praktische info).
Route genereren en op kaart tonen met polyline + stops.
Story genereren en tonen voor een Gem.
Allowlisted frontend analytics-events veilig doorgeven zonder chatcontent op te slaan.
Admin-auth en read-only analytics-data veilig laden voor het analytics portal.
Errors zijn consistent (400/404/429/5xx) en in lijn met NFR’s (geen leak van interne details).
Contracts matchen FR/NFR + Gherkin in 2 Specificaties en sluiten aan op docs/mcd.md.
