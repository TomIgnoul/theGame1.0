# 1.2 Scope IN/OUT

## Doel van deze scope

Deze scope bepaalt wat “MVP” betekent voor dit project. Alles wat **IN-scope** staat moet af zijn om de MVP als “done” te kunnen demonstreren. Alles wat **OUT-scope** staat is bewust uitgesteld om scope creep te vermijden en de kernflow eerst stabiel te krijgen.

---

## Scope IN (MVP)

### A. Kaart & ontdekking

- Interactieve kaart van Brussel met pins op basis van seed data (openadata.brussels POI’s).
- Filteren op thema (MVP): **War, Museum, Streetart, Food, Culture**.
- Een POI kunnen selecteren vanuit de kaart om details te bekijken.

### B. Detailpagina per POI

- Detailpagina met minimaal:
    - naam/titel
    - locatie/adres (indien beschikbaar)
    - korte AI-story (op basis van beschikbare data)
    - basis praktische info (alleen wat in dataset beschikbaar is)

### C. Route builder (kern van de MVP)

- Gebruiker kiest **thema + tijd en/of afstand (km)**.
- Route wordt gegenereerd en getoond met:
    - route-lijn op de kaart
    - genummerde stops
    - “Start wandeling” flow (navigatie-ready)

### D. AI gids (MVP-variant met guardrails)

- Chat ondersteunt:
    - storytelling over de geselecteerde POI
    - praktische info enkel als die in de dataset zit
    - wanneer data ontbreekt: “niet gevonden/onzeker” i.p.v. gokken (anti-hallucinatie)

### E. Dataset MVP (bewuste beperking)

- Hard cap: **5 POI’s** om filters + routing realistisch te kunnen testen zonder content overload.

---

## Scope OUT (Post-MVP / Roadmap)

### A. Onboarding van “parels” (post-launch)

- “Parel aanmelden” (formulier + review + publiceren)
- Moderatie/validatie workflow voor kwaliteit en anti-spam

### B. Content & media uitbreidingen

- Volwaardige audiogids pipeline (TTS + voice management)
- Multi-language ondersteuning
- Uitgebreide media (video, galerijen, rijke storytelling formats)

### C. Data & platform

- Scraping (tenzij open data onvoldoende blijkt)
- Airflow/automation pipelines
- User accounts, favorieten, ratings/reviews
- Analytics dashboard

### D. UX polish (bewust uitgesteld)

- Advanced animaties, micro-interactions, gamification
- Social sharing flows en extra “nice-to-have” UI features

---

## MVP Gate (wanneer is dit “done”?)

De MVP is pas “done” wanneer:

1. Route correct verschijnt als **lijn met genummerde stops** en startbaar is via één duidelijke flow.
2. AI chat **geen hallucinaties** levert: info komt uit dataset of wordt expliciet als “niet gevonden/onzeker” gelabeld.
3. Kernflow werkt end-to-end: **kaart → selectie → details → route → start wandeling** zonder dead ends.

---

## Rationale (waarom deze scope logisch is)

- De route builder is de kernwaarde: zonder route is er geen “product”.
- We beperken dataset en polish om integratierisico en tijdsdruk te beheersen.
- AI wordt gebruikt als versneller, maar met guardrails om kwaliteit te garanderen.
