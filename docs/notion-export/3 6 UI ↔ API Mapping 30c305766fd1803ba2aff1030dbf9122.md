# 3.6 UI ↔ API Mapping

## Doel

Deze mapping legt vast **welke UI actie** welke **backend endpoint** oproept, met verwachte states (loading/empty/error). Dit is een design-artefact (geen implementatie) en is bedoeld om build friction te minimaliseren (ook voor AI agents).

---

## Conventies

- Base URL (local): `http://localhost:8000`
- Alle endpoints hieronder zijn relatief aan `/api`.
- Error handling (UI):
  - `400` = input invalid → inline validatie en/of foutmelding bij formulier.
  - `404` = resource niet gevonden → duidelijke boodschap.
  - `429` = rate limit → tekst: **“Even wachten en opnieuw proberen.”**
  - `5xx` / `502` / `503` = backend error → error banner + retry mogelijkheid.

## Story Prompt (conceptueel template)

Frontend stuurt conceptueel:

> Generate a short story about this POI (max 120 words).  
> Use only known facts; if unsure, say you are unsure.

Backend:

- Verrijkt de prompt met Gem‑context (`title`, `theme`, locatie, praktische info).
- Respecteert max lengte en “niet hallucineren” constraint.
- Slaat alleen de uiteindelijke `storyText` op in `gem_stories` (zie 3.2).

## UI Proof Points (acceptatiebewijs)

- Filter badge zichtbaar wanneer filter actief is (S1).
- Route view toont:
  - Polyline
  - Genummerde stops (Stop 1..N) die overeenkomen met de lijst (S3/S4).
- Detailpagina toont bij ontbrekende praktische info het label:
  - **“Praktische info niet gevonden”**.
- Story‑sectie toont duidelijke loading/error states en herprobeer mogelijkheid.
- Bij 429 wordt tekst **“Even wachten en opnieuw proberen.”** getoond in de chat/story UI.
- Admin portal toont bij lege dataset zero-state KPI’s en lege grafieken in plaats van een backend error.

## Security hooks (design)

- `POST /api/gems/:id/story`:
  - Max inputlengte voor `message` (indien gebruikt) = **500 chars**.
  - Rate limiting actief (config in backend).
- `POST /api/routes`:
  - Rate limiting actief om kosten van routing onder controle te houden.
- `POST /api/admin/auth/login`:
  - Gebruikt gedeelde env-passphrase server-side.
  - Geeft een httpOnly cookie uit; frontend leest het token niet direct.
- `GET /api/admin/analytics/*`:
  - Read-only endpoints.
  - Vereisen geldige admin-sessie via cookie.
- Geen persistente opslag van chatberichten of volledige prompts/responses:
  - Alleen `gem_stories` en optionele `route_logs` bevatten afgeleide of geaggregeerde data.
